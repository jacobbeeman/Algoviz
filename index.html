Algorithm Descriptions

Short and simple description to turn code and diagrams into understandable information

Bubble Sort

A simple sorting algorithm that works by repeatedly comparing pairs of neighboring elements and swapping them if they are in the wrong order. This process continues until no more swaps are needed, meaning the list is sorted. The largest values “bubble up” to the end of the list after each pass, which is how it gets its name. While easy to understand and great for learning, Bubble Sort is not very efficient for large datasets because it has to make many comparisons and swaps.

Big O complexities

  • Best Case: O(n)
  • Average Case:O(n2)
  • Worst Case: O(n2)

Insertion Sort

A straightforward sorting algorithm that builds a sorted list one element at a time. It works by taking one item from the unsorted part of the list and inserting it into its correct position in the sorted part, just like sorting playing cards in your hand. This process repeats until all elements are in order. Insertion Sort is simple to understand and efficient for small or nearly sorted datasets, but it becomes slower when working with large amounts of data.

Big O complexities

  • Best Case: O(n)
  • Average Case:O(n2)
  • Worst Case: O(n2)

Binary search

A efficient algorithm used to find a specific value in a sorted list. It works by repeatedly dividing the list in half and comparing the target value to the middle element. If the target is smaller, the search continues in the left half; if it’s larger, it continues in the right half. This process repeats until the value is found or the list can’t be divided any further. Binary Search is much faster than checking each element one by one, but it only works correctly on data that is already sorted.

Big O complexities

  • Best Case: O(1)
  • Average Case:O(log n)
  • Worst Case: O(log n)

Quick Sort

A fast and efficient sorting algorithm that works by selecting a pivot element and dividing the list into two smaller lists — one with values less than the pivot and one with values greater than the pivot. It then repeats this process (recursively) on the smaller lists until everything is sorted. Quick Sort is known for its divide-and-conquer approach and usually runs very quickly, especially on large datasets.

Big O complexities

  • Best Case: O(n log n)
  • Average Case:O(n log n)
  • Worst Case: O(n2)

Linear Search

A basic search algorithm that looks for a specific value by checking each element in a list, one by one, from start to finish. If it finds the value, it stops and returns the position; if not, it goes through the entire list. Linear Search is easy to understand and works on any list, whether it’s sorted or not, but it can be slow for very large datasets.

Big O complexities

  • Best Case: O(1)
  • Average Case:O(n)
  • Worst Case: O(n)

Python Examples

Bubble Sort

def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]

Insertion Sort

def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i-1
        while j >= 0 and arr[j] > key:
            arr[j+1] = arr[j]
            j -= 1
        arr[j+1] = key

Binary Search

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

Quick Sort

def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr)//2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + middle + quick_sort(right)

Linear Search

def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

Binary Search

int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while(left <= right) {
        int mid = left + (right - left) / 2;
        if(arr[mid] == target)
            return mid;
        else if(arr[mid] < target)
            left = mid + 1;
        else
            right = mid - 1;
    }
    return -1;
}

Quick Sort

void quickSort(int[] arr, int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

int partition(int[] arr, int low, int high) {
    int pivot = arr[high];
    int i = (low - 1);
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }
    int temp = arr[i + 1];
    arr[i + 1] = arr[high];
    arr[high] = temp;
    return i + 1;
}

            return i;

}

function binarySearch(arr, target) {
    let left = 0, right = arr.length - 1;
    while(left <= right) {
        let mid = Math.floor((left + right) / 2);
        if(arr[mid] === target)
            return mid;
        else if(arr[mid] < target)
            left = mid + 1;
        else
            right = mid - 1;
    }
    return -1;
}

Quick Sort

function quickSort(arr) {
    if (arr.length <= 1) return arr;
    const pivot = arr[arr.length - 1];
    const left = arr.filter(x => x < pivot);
    const right = arr.filter(x => x > pivot);
    const equal = arr.filter(x => x === pivot);
    return [...quickSort(left), ...equal, ...quickSort(right)];
}

            return i;
    }

Glossary

Quick definitions to help new users understand common terms used in AlgoViz.

Algorithm

A step-by-step set of instructions to perform a task such as sorting or searching.

Binary search

A search algorithm that efficiently finds a target value in a sorted dataset by repeatedly dividing the search range in half.

Bubble Sort

A simple sorting algorithm that compares and swaps adjacent elements until the dataset is sorted.

Complexity

A measure of the efficiency of an algorithm, usually expresses using expressed using Big O notation (e.g., O(n), O(n^2)). It indicates how the algorithm’s performances change as the dataset size increases in terms of time or space usage.

Dataset

A structured collection of data values that an algorithm processes. In Algoviz, datasets are visually represented (such as bars or nodes) and are sorted or searched to demonstrate how algorithms function.

Dot Nodes

Small circular graphical elements that represent individual data points in a dataset, often used in visualizations of searching algorithms or graph structures.

Efficient Big O

A measure of an algorithm’s efficiency, describing how its runtime or space requirements grow as the input size increases.

Element

An individual item or value in a dataset, or a building block in an HTML document or data structure.

Insertion Sort

A sorting algorithm that builds a sorted array one item at a time by inserting elements into their correct position.

Node

A basic unit of a data structure, such as a linked list or tree, that contains data and links to other nodes.

Recursive

A function or algorithm that calls itself repeatedly until a base condition is met.
